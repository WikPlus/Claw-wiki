Zero-Latency Audio Pipeline

The default OpenClaw audio handling often suffers from a slow "decide-download-transcribe-reply" loop that can take up to 8-10 seconds. This protocol bypasses the agent-thinking cycle for transcription entirely.

The Logic

By configuring the cli type transcription model in the gateway config, the system performs the transcription as a pre-processing step. The agent receives the text immediately, cutting the round-trip time significantly.

1. The Script (fast_transcribe.sh)

Place this in your skills folder. It uses raw curl to Groq for maximum speed.

#!/usr/bin/env bash
API_KEY="YOUR_GROQ_KEY"
URL="https://api.groq.com/openai/v1/audio/transcriptions"
FILE="$1"

# Fast execution
RESPONSE=$(curl -s "$URL" \
  -H "Authorization: Bearer $API_KEY" \
  -F "file=@$FILE" \
  -F "model=whisper-large-v3-turbo" \
  -F "response_format=json")

echo "$RESPONSE" | jq -r '.text' | tr -d '\n'

2. The Config

Update your openclaw.json to use the new script. Ensure maxBytes is high enough to allow file reading.

"audio": {
  "enabled": true,
  "maxBytes": 25000000,
  "models": [{
    "type": "cli",
    "command": "/path/to/fast_transcribe.sh",
    "args": ["{{MediaPath}}"]
  }]
}
